# MST Guesser — 最小全域木を当てろ！

**視覚直感 vs 数学アルゴリズム** を体感するゲーム
ランダムに配置された点を「すべてつなげ」「ループ禁止」「できるだけ短く」を満たすように直感で辺を選び、正解の最小全域木（MST）と比較してスコアを競う

---

## ゲームの流れ

1. **タイトル画面** — ルール確認 → 難易度選択（初級／中級／上級）→ スタート
2. **プレイ画面** — 点をクリックして辺を引く（既存の辺をもう一度選ぶと削除）
3. **結果画面** — 正解MSTと比較、色分け表示、スコア表示
4. **最終画面** — 全5問の「MST直感指数」、結果コピー、再プレイ

---

## アルゴリズム・データ構造

### 1. 点の生成 `genPoints(n)`

**目的**: 重ならないように n 個の点をボード内にランダム配置する。

**手順**:
- リジェクションサンプリング: ランダムな (x, y) を生成
- 既存の点すべてとの距離が `MIN_D` 以上か判定
- 条件を満たす点のみ追加
- 3000 試行以内に n 個揃わなければやり直し

**計算量**: 試行回数に依存。点が増えるほど試行回数が増える可能性あり。

---

### 2. 最小全域木（MST）計算 `computeMST(pts)`

**アルゴリズム**: **Kruskal 法**

**目的**: 全点を連結する、重み合計が最小の辺集合を求める。

**手順**:
1. **辺リスト構築**: 全点対 (i, j) について `{ i, j, d: dist(pts[i], pts[j]) }` を生成
2. **辺のソート**: 距離 `d` の昇順にソート
3. **Union-Find（素集合）で辺を追加**:
   - ソート順に辺を 1 本ずつ見る
   - 両端点が同じ連結成分に属していなければ、その辺を採用し、2 つの成分を `union`
   - 既に同じ成分ならスキップ（ループ防止）
   - 採用した辺が `n-1` 本になったら終了

**Union-Find**:
- `find(x)`: 経路圧縮（path compression）で根を返す
- `union(x, y)`: ランク（木の高さ）でマージし、平衡を保つ

**計算量**: 辺数 O(n²)、ソート O(n² log n)、Union-Find はほぼ定数オーダー。全体で **O(n² log n)**。

---

### 3. サイクル検出 `wouldCycle(edges, a, b, n)`

**目的**: 辺 (a, b) を追加するとループになるか判定する。

**アルゴリズム**: **深さ優先探索（DFS）**

**手順**:
1. `edges` から隣接リスト `adj` を構築
2. ノード `a` からスタックベースの DFS を開始
3. `b` に到達すれば「既に a と b は連結」→ 辺を追加するとサイクル発生
4. 到達できなければサイクルは発生しない

**計算量**: **O(n + 辺数)**。辺は高々 n-1 本なので O(n)。

---

### 4. 距離計算 `dist(a, b)`

ユークリッド距離:

```
dist = √((a.x - b.x)² + (a.y - b.y)²)
```

---

### 5. 辺の一意キー `edgeKey(i, j)`

`i < j` なら `"i,j"`、そうでなければ `"j,i"` を返す。  
無向辺の重複判定・集合管理に使用。

---

### 6. スコア計算 `score( playerCost, mstCost )`

```
score = min( 100, (mstCost / playerCost) × 100 )
```

- 100: 完璧（MST と一致）
- 95 以上: かなり良い
- 90 以下: 改善の余地あり

## 動作メソッド一覧

### 状態管理 (`state.js`)

| メソッド | 説明 |
|----------|------|
| `freshState()` | 初期状態オブジェクトを返す |

### ゲーム制御 (`game.js`)

| メソッド | 説明 |
|----------|------|
| `startGame()` | 状態リセット → 第 1 ラウンド開始 |
| `nextRound()` | ラウンド番号を進め、点・MST を生成しプレイ画面へ |
| `handleNode(idx)` | ノードクリック時の処理。選択／辺の追加／辺の削除（既存の辺を再選択すると削除） |
| `undoEdge()` | 直前に追加した辺を 1 本削除 |
| `resetEdges()` | そのラウンドの辺をすべて消してやり直し |
| `submitAnswer()` | 回答として受理し、スコア計算 → 結果画面へ |
| `goNext()` | 次のラウンドへ、または最終画面へ |
| `copyResult()` | 結果テキストをクリップボードにコピー |

### 描画 (`render.js`)

| メソッド | 説明 |
|----------|------|
| `showTitle()` | タイトル・ルール・スタートボタンを表示 |
| `showPlay()` | プレイ画面を組み立て、ボード・ボタン・イベントを設定 |
| `renderBoard()` | プレイ中の SVG ボードを再描画（辺・プレビュー線・ノード） |
| `showResult(score, pc, mc)` | 結果画面（正解/誤りの色分け、スコア、コスト表示） |
| `showFinal()` | 最終画面（MST直感指数、ラウンド別スコア、コピー・リトライ） |

### ユーティリティ

| メソッド | 説明 |
|----------|------|

## 難易度

| 難易度 | 点の数 |
|--------|--------|
| 初級   | 5〜6 点 |
| 中級   | 7〜9 点 |
| 上級   | 9〜12 点 |

## 使い方

1. `index.html` をブラウザで開く
2. 難易度（初級／中級／上級）を選択
3. 点をクリックして辺を引く（もう一度同じ 2 点を選ぶと辺が消える）
4. 「回答する」で正解と比較
5. 5 問終了後、結果をコピーしてシェア可能
